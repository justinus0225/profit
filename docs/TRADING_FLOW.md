# P.R.O.F.I.T. 실제 매매 운영 흐름

## 24시간 자동매매 시스템의 동작 상세

---

# 1. 자금 구조: "100만 USDT가 있을 때"

시스템이 매매에 사용하는 자금의 구조를 먼저 정의한다.
모든 비율은 관리자가 변경 가능하며, 아래는 기본값이다.

```
총 자산: 1,000,000 USDT (예시)
│
├── [Reserve] 최소 보유금: 300,000 USDT (30%) ─── 절대 매매에 사용 안 함
│
└── [Available] 가용 투자금: 700,000 USDT (70%)
     │
     ├── 리스크 레벨에 따라 실제 사용 가능 금액이 줄어듦
     │
     │   리스크 스코어  0~30  → 가용금의 100% = 700,000 USDT 사용 가능
     │   리스크 스코어 31~60  → 가용금의  70% = 490,000 USDT 사용 가능
     │   리스크 스코어 61~80  → 가용금의  40% = 280,000 USDT 사용 가능
     │   리스크 스코어 81~100 → 신규 매매 중단. 기존 포지션 단계적 청산
     │
     └── 단일 코인 최대 한도: 실제 사용 가능 금액의 20%
         (리스크 Low일 때: 700,000 × 20% = 140,000 USDT)
```

### 1.1. 자금 사용 예시: 리스크 스코어 45 (Medium) 일 때

```
총 자산:       1,000,000 USDT
최소 보유금:      300,000 USDT (30%, 매매 불가)
가용 투자금:      700,000 USDT (70%)
리스크 조정:      700,000 × 0.7 = 490,000 USDT (실제 투자 가능)
단일 코인 한도:   490,000 × 0.2 = 98,000 USDT (한 코인 최대)
현재 투자 중:     250,000 USDT (BTC 120K, ETH 80K, SOL 50K)
추가 투자 가능:   490,000 - 250,000 = 240,000 USDT
```

### 1.2. 분할 매수 규칙

하나의 매수 시그널이 발생하면 **한 번에 전량 매수하지 않는다**.

| 분할 | 비율 | 조건 |
|------|------|------|
| 1차 매수 | 배정금의 40% | 시그널 발생 즉시 |
| 2차 매수 | 배정금의 30% | 1차 매수가 대비 -2% 하락 시 (더 좋은 가격에 추가) |
| 3차 매수 | 배정금의 30% | 1차 매수가 대비 -5% 하락 시 (최종 물타기) |

1차 매수 후 가격이 상승하면 2차/3차 매수는 실행하지 않는다.
이 경우 남은 배정금은 가용 자금으로 복귀한다.

# 2. 에이전트별 24시간 동작 주기

시스템이 켜진 순간부터 모든 에이전트가 각자의 주기로 독립 동작한다.

```
시간축 (1시간 단위)
00  01  02  03  04  05  06  07  08  09  10  11  12 ...
│   │   │   │   │   │   │   │   │   │   │   │   │
│───────────────────────────────────────────────────
│ 데이터 엔지니어: WebSocket 실시간 가격 스트림 수신 (항상 동작)
│───────────────────────────────────────────────────
│ 리스크 관리: 보유 포지션 손익 실시간 감시 (항상 동작)
│───────────────────────────────────────────────────
│ 매매 실행: 체결 상태 폴링, 서킷 브레이커 감시 (항상 동작)
│───────────────────────────────────────────────────
│   │   │   │   │   │   │   │   │   │   │   │   │
├───┼───┼───┼───┼───┼───┼───┼───┼───┼───┼───┼───┤
│ ▲ │ ▲ │ ▲ │ ▲ │ ▲ │ ▲ │ ▲ │ ▲ │ ▲ │ ▲ │ ▲ │ ▲ │
│ 퀀트: 15분 마다 전체 Watchlist 기술적 지표 스캔
│───────────────────────────────────────────────────
│       │       │       │       │       │       │
│       ▲       ▲       ▲       ▲       ▲       ▲
│ 퀀트: 1시간 마다 멀티 타임프레임 종합 분석
│───────────────────────────────────────────────────
│               │               │               │
│               ▲               ▲               ▲
│ 경제 분석: 4시간 마다 거시환경 + 시장 심리 업데이트
│───────────────────────────────────────────────────
│                                               │
│ 경제 분석: 1일 1회 Watchlist 갱신 + 펀더멘탈 재평가
│ 리스크 관리: 1일 1회 리스크 스코어 전체 재산출
│ 포트폴리오: 1일 1회 리밸런싱 점검
│───────────────────────────────────────────────────
```

## 2.1. 상시 동작 에이전트 (Always-On)

이 에이전트들은 이벤트 루프 형태로 끊임없이 동작한다.

| 에이전트 | 동작 방식 | 하는 일 |
|----------|-----------|---------|
| **데이터 엔지니어** | WebSocket 실시간 | 거래소 가격 스트림을 TimescaleDB에 적재. 틱/분봉/시봉 자동 집계 |
| **리스크 관리** | 10초 간격 폴링 | 보유 포지션의 현재 손익 계산. 손절선/트레일링 스탑 도달 여부 감시 |
| **매매 실행** | 이벤트 대기 | 매매 지시 수신 시 주문 실행. 미체결 주문 상태 폴링(30초). 서킷 브레이커 감시 |
| **오케스트레이터** | 이벤트 대기 | 에이전트 간 메시지 라우팅. 합의 프로세스 조율. OpenClaw 메시지 수신 |

## 2.2. 주기적 동작 에이전트 (Scheduled)

| 에이전트 | 주기 | 하는 일 |
|----------|------|---------|
| **퀀트 (빠른 스캔)** | 매 15분 | Watchlist 전체 코인의 기술적 지표(RSI, MACD, BB) 빠른 연산. 임계값 돌파 감지 |
| **퀀트 (깊은 분석)** | 매 1시간 | 멀티 타임프레임(15m, 1H, 4H, 1D) 종합 분석. 매매 시그널 스코어링 |
| **퀀트 (전략 평가)** | 매 4시간 | 활성 전략의 최근 성과 평가. 시그널 임계값 미세 조정 |
| **경제 분석 (시장 심리)** | 매 4시간 | Fear & Greed Index, BTC Dominance, 스테이블코인 유입량 갱신 |
| **경제 분석 (뉴스)** | 매 1시간 | 뉴스/SNS 크롤링. NLP 감성 스코어 갱신 |
| **경제 분석 (거시)** | 매일 06:00 UTC | 거시 경제 지표 전체 재평가. 시장 방향성 리포트 갱신 |
| **경제 분석 (유니버스)** | 매일 00:00 UTC | Watchlist 전체 갱신: 시가총액 순위, 거래량, 토큰 언락, 상장/폐지 |
| **리스크 관리 (전체 평가)** | 매일 00:00 UTC | 리스크 스코어 전체 재산출. 가용 투자금 한도 재설정 |
| **포트폴리오 (리밸런싱)** | 매일 00:00 UTC | 보유 코인 비중 점검. 단기 만기 도래 포지션 연장/청산 결정 |
| **포트폴리오 (성과)** | 매일 09:00 UTC | 전일 매매 성과 리포트 생성. 관리자에게 OpenClaw로 발송 |

## 2.3. 이벤트 기반 동작 (Event-Driven)

정기 스캔 외에, 아래 이벤트가 감지되면 **즉시** 관련 에이전트가 기동한다.

| 이벤트 | 감지 주체 | 트리거 조건 | 반응 |
|--------|-----------|-------------|------|
| 급등/급락 | 데이터 엔지니어 | 5분 내 ±3% 가격 변동 | 퀀트에게 긴급 분석 요청 |
| 대량 거래 | 데이터 엔지니어 | 평균 거래량의 5배 이상 단일 체결 | 경제 분석에 알림 |
| 뉴스 속보 | 경제 분석 | 주요 키워드 감지 (규제, 해킹, 상장 등) | 관련 코인 긴급 재평가 |
| 손절선 도달 | 리스크 관리 | 보유 코인이 손절가 도달 | 즉시 매도 지시 (합의 생략) |
| 트레일링 스탑 | 리스크 관리 | 보유 코인이 고점 대비 -3% 하락 | 즉시 매도 지시 (합의 생략) |
| 일일 손실 한도 | 리스크 관리 | 당일 실현+미실현 손실 합계 ≥ -3% | 당일 신규 매매 전체 중단 |
| 관리자 명령 | 오케스트레이터 | OpenClaw 메시지 수신 | 명령에 따라 해당 에이전트 기동 |

# 3. 코인 선별: 어떻게 투자할 코인을 고르는가

## 3.1. 일일 유니버스 갱신 (매일 00:00 UTC)

경제 분석 에이전트가 매일 자정에 투자 가능 코인 목록(Watchlist)을 새로 생성한다.

```
[Step 1] 전체 코인 목록 가져오기
  CoinGecko/CMC API → 전체 코인 (수천 개)
         │
         ▼
[Step 2] 1차 필터 — 유동성 기준
  │
  ├── 시가총액 Top 100 이내?              → NO → 제외
  ├── 24H 거래량 $10M 이상?              → NO → 제외
  ├── 타겟 거래소(예: Binance)에 상장?     → NO → 제외
  └── 통과: 약 40~60개 코인
         │
         ▼
[Step 3] 2차 필터 — 리스크 제거
  │
  ├── 블랙리스트에 등록된 코인?            → YES → 제외
  ├── 30일 내 유통량 5%+ 대규모 토큰 언락? → YES → 경고 플래그 (제외는 아님)
  ├── 최근 해킹/보안 사고?                → YES → 제외
  ├── 규제 이슈(SEC 소송 등)?             → YES → 경고 플래그
  └── 통과: 약 30~50개 코인
         │
         ▼
[Step 4] 펀더멘탈 스코어 산출 (0~100)
  │
  ├── 시가총액 순위 점수     (20점)  Top 10=20, Top 30=15, Top 50=10, Top 100=5
  ├── 거래량/시총 비율       (15점)  높을수록 유동성 우수
  ├── 온체인 활성도          (15점)  활성 주소, 트랜잭션 수
  ├── 개발 활동 (GitHub)    (15점)  최근 30일 커밋, 컨트리뷰터 수
  ├── 커뮤니티 감성          (15점)  NLP 긍정/부정 점수
  ├── 토큰 이코노미          (10점)  인플레이션율, 소각 메커니즘
  └── 거래소 유동성          (10점)  호가 스프레드, 오더북 깊이
  │
  └── 결과: Watchlist (펀더멘탈 스코어 40점 이상만)
            약 20~35개 코인
            + 코인별 펀더멘탈 스코어
            + 경고 플래그 (토큰 언락, 규제 등)
```

## 4.2. 매수 시그널 발생 조건 요약

퀀트 에이전트가 시그널을 생성하는 기술적 조건:

### 전략별 매수 트리거

| 전략 | 트리거 조건 | 보유 기간 | 시그널 스코어 가중치 |
|------|-----------|-----------|---------------------|
| **평균 회귀** | RSI ≤ 30 + BB 하단 이탈 + MACD 수렴 | 단기 (1~7일) | RSI 40%, BB 30%, MACD 30% |
| **추세 추종** | MA20 > MA50 골든크로스 + ADX ≥ 25 + 거래량 증가 | 중기 (1~4주) | MA 35%, ADX 35%, 거래량 30% |
| **모멘텀** | 5분 내 +3% 급등 + 거래량 5배 급증 + RSI 50~70 | 단기 (1~3일) | 가격변동 40%, 거래량 40%, RSI 20% |
| **브레이크아웃** | 20일 고점 돌파 + ATR 확대 + 거래량 동반 | 중기 (1~4주) | 돌파강도 40%, ATR 30%, 거래량 30% |

### 시그널 스코어 해석

```
시그널 스코어 범위: -100 ~ +100

  +100 ──── 극강 매수 (모든 지표 일치, 극히 드묾)
  + 80 ──── 강한 매수 (대부분 지표 일치)
  + 50 ──── 매수 임계값 (이 이상이면 시그널 발생)
       ...
     0 ──── 중립 (매매 시그널 없음)
       ...
  - 50 ──── 매도 임계값 (보유 중인 코인에 대해)
  - 80 ──── 강한 매도
  -100 ──── 극강 매도

  ±50 미만: 시그널 생성하지 않음 (노이즈로 간주)
```

## 4.3. 매수가 기각되는 경우

시그널이 생성되어도 합의 과정에서 기각될 수 있다.

| 기각 사유 | 기각 주체 | 예시 |
|-----------|-----------|------|
| 거시 환경 불일치 | 경제 분석 | 퀀트는 BUY인데, 경제분석이 "강한 하락장 진입" 판단 |
| 코사인 유사도 미달 | 오케스트레이터 | 퀀트 방향과 경제분석 방향의 유사도 < 0.6 |
| 자금 부족 | 리스크 관리 | 가용 투자금 잔여분 < 요청 배정금 |
| 단일 코인 한도 초과 | 리스크 관리 | 해당 코인에 이미 한도만큼 투자 중 |
| 일일 손실 한도 도달 | 리스크 관리 | 당일 -3% 손실 이미 도달 |
| 리스크 레벨 Critical | 리스크 관리 | 리스크 스코어 81 이상, 전체 매매 중단 |
| 서킷 브레이커 활성 | 매매 실행 | 연속 손실, API 오류 등으로 매매 정지 상태 |
| 포트폴리오 상관성 | 포트폴리오 관리 | 기존 보유 코인과 상관계수가 너무 높음 |

# 5. 매도: 언제, 왜 파는가

보유 중인 코인을 매도하는 경우는 크게 **5가지 경로**가 있다.

## 5.1. 경로 1: 목표가 도달 (이익 실현)

```
[리스크 관리 에이전트: 10초 간격 감시]

  BTC 매수가:   64,795 USDT
  현재가:       67,250 USDT (+3.79%)
  목표가:       67,200 USDT (+3.7%) ← 도달!

  → 목표가 도달 감지
  → 즉시 매도 지시 생성 (합의 불필요, 사전 승인된 이익 실현)
  → 매매 실행 에이전트: 시장가 매도 실행
  → 수익: +3.79% (수수료 차감 전)

  매도 방식: 전량 매도 (목표가 도달 = 전략 완료)
```

## 5.2. 경로 2: 손절가 도달 (손실 제한)

```
[리스크 관리 에이전트: 10초 간격 감시]

  BTC 매수가:   64,795 USDT
  현재가:       63,480 USDT (-2.03%)
  손절가:       63,500 USDT (-2.0%) ← 도달!

  → 손절 트리거 발동
  → 합의 생략 — 즉시 매도 (리스크 관리 에이전트의 단독 권한)
  → 매매 실행 에이전트: 시장가 매도 즉시 실행
  → 손실: -2.03% (수수료 차감 전)

  매도 방식: 전량 매도 (손절 = 무조건 전량)
  분할 매수 2차/3차 예약: 자동 취소
```

**손절은 합의 프로세스를 거치지 않는다.** 가격이 손절선에 도달하면 리스크 관리 에이전트가
단독 권한으로 즉시 매도를 실행한다. 합의를 기다리는 사이에 손실이 확대될 수 있기 때문이다.

## 5.3. 경로 3: 트레일링 스탑 (수익 보호)

```
[시나리오: BTC가 상승 후 하락 반전]

  매수가: 64,795 USDT
  트레일링 스탑: 고점 대비 -3%

  시간 경과:
  ──────────────────────────────────────────
  Day 1:  64,795 → 65,500 (고점 갱신)       트레일링 기준: 65,500 × 0.97 = 63,535
  Day 2:  65,500 → 66,800 (고점 갱신)       트레일링 기준: 66,800 × 0.97 = 64,796
  Day 3:  66,800 → 67,500 (고점 갱신!)      트레일링 기준: 67,500 × 0.97 = 65,475
  Day 4:  67,500 → 66,200 (하락 시작)       트레일링 기준: 67,500 × 0.97 = 65,475
  Day 4:  66,200 → 65,400 (계속 하락)       65,475 도달! → 트레일링 스탑 발동!
  ──────────────────────────────────────────

  → 매도 실행: 65,400 USDT
  → 수익: +0.93% (고점 대비 -3.1%이지만 원금 대비는 수익)
  → 합의 생략: 리스크 관리 에이전트 단독 권한

  트레일링 스탑의 핵심:
  "고점이 올라갈수록 손절선도 따라 올라간다"
  → 수익을 보호하면서 상승 추세를 최대한 따라간다
```
